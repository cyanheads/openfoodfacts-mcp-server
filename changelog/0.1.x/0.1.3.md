# 0.1.3 — 2026-06-04

## Added

- **`off_search_products` `sort_by` parameter** — optional enum (`last_modified_t`, `unique_scans_n`, `created_t`, `popularity_key`) passed through to the upstream `/api/v2/search` `sort_by` query param on the tag-filter path. Ignored on text-query searches (search.openfoodfacts.org does not support server-side sort). ([#2](https://github.com/cyanheads/openfoodfacts-mcp-server/issues/2))
- **`ecoscore_grade`** added to `SEARCH_FIELDS`, `SearchParams`, `RawTextSearchHit`, the output schema, the text-search normalizer, and `format()` (`Green-Score: <grade>` alongside Nutri-Score / NOVA). ([#2](https://github.com/cyanheads/openfoodfacts-mcp-server/issues/2))
- **21 new tests** covering `sort_by` pass-through (tag-filter URL), `sort_by` absent from text-search URL, `ecoscore_grade` in result rows (present and absent), service-layer URL and text-search normalization, and `format()` rendering.
