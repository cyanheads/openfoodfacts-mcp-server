# 0.1.4 — 2026-06-12

## Added

- **Truncation disclosure** — `off_search_products` and `off_browse_taxonomy` now emit `ctx.enrich.truncated(...)` with `truncated` / `shown` / `cap` output fields when the response is paginated or capped. Search includes a `guidance` string naming the current page and total page count (`Page N of M. Use page parameter to fetch subsequent pages.`).
- **Explicit server identity** — `createApp()` sets `name` and `title` to `openfoodfacts-mcp-server` so the display identity is the unscoped repo name across all client surfaces, rather than falling back to the scoped package name.
- **`scripts/clean-mcpb.ts`** — post-pack bundle cleaner wired into the `bundle` script: runs `mcpb clean` (dev-dependency prune + manifest validation), then strips dependency-shipped agent-doc entries (`skills/`, `.claude/`, `.agents/`, `SKILL.md`) nested under `node_modules/` that root-anchored `.mcpbignore` patterns cannot reach, and asserts zero remain.
- **Docker `HEALTHCHECK`** — polls `/healthz` every 30s via `bun -e fetch` (no curl/wget dependency); OCI `org.opencontainers.image.version` label populated from a new `APP_VERSION` build arg.
- **`techniques` skill** — catalog of reusable response/data-shaping patterns (overflow handling, payload shaping), seeded with the outline-on-overflow reference.

## Changed

- **`off_search_products` `no_filters` error** — code changed from `InvalidParams` to `ValidationError`.
- **`.mcpbignore`** — root dev-dir patterns anchored (`skills/` → `/skills/`, `.claude/` → `/.claude/`, `.agents/` → `/.agents/`) so they no longer also strip nested runtime paths like `node_modules/<pkg>/skills/`.
- **`scripts/lint-packaging.ts`** — adds bundle-content guards (unanchored root dev-dir patterns, stripped critical runtime paths, post-bundle `node_modules/**` agent-doc entries) and an identity check that `createApp()` `name`/`title` and manifest `display_name` equal the unscoped package name.
- **`scripts/check-framework-antipatterns.ts`** — new `coerce-boolean-env-flag` rule flags `z.coerce.boolean()` on env flags (`Boolean("false")` is `true` — use `z.stringbool()`); comment lines are skipped so documenting the pattern doesn't trip the scan.
- **Vendored skills** re-synced to the installed framework version under `skills/`, `.claude/skills/`, and `.agents/skills/`.
